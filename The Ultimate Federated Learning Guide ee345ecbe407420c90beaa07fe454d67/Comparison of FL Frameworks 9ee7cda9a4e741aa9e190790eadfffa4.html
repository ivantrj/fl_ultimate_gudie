Comparison of Federated Learning technologies

PySyft 0.3

  • Duet can be used only in real-time, which means that on both sides (data owner and data scientist) they have to sit in front of their laptop during analysis, since every time the data scientist wants to get data (even it's a calculated value, e.g. mean) he needs to send a request to the data owner and the data owner has to accept it. So this is an "online" training and analysis. In short term, they assume that data owner and data scientist share their screens during analysis.
  • They plan to create offline training, where the data owner doesn't have to accept every request, but they don't know exactly how to solve it yet.
  • They work with an "allowlist", which means we can't use a PyTorch function remotely until they add it to the "allowlist". That is necessary for them to make remote execution secure.

Here are the sources:

  • Roadmap: https://www.youtube.com/watch?v=QUTRN3G1l0I&feature=youtu.be&t=310
  • Duet tutorial: https://www.youtube.com/watch?v=DppXfA6C8L8&ab_channel=OpenMined

We shouldn't use PySyft 0.3, since for now, it's inflexible and not scalable with Duet. On the other hand, we can't wait for them to allow the remote execution of Pytorch functions, and the offline training is missing, which is essential. Their roadmap is rough and living so it can change anytime. Moreover, as this version is under development, it may contain bugs even later on, and we would have to wait for them to fix it.

Tensorflow Federated

https://www.tensorflow.org/federated/faq

TensorFlow Federated (developed by Google) speciﬁcally targets research use cases, providing large-scale simulation capabilities as well as ﬂexible orchestration for the control of sampling. [Kairouz, 2019]

TFF is an open-source framework for machine learning and other computations on decentralized data. TFF has been developed to facilitate open research and experimentation with Federated Learning.

Amond TFF’s case studies we can find health care related case studies as well:

  • Sinovation Ventures: Disease classification and segmentation were performed on retinal Optical coherence tomography (OCT) images with the use of TFF.
  • GE Healthcare: With TFF GE healthcare is training a neural network to identify specific anatomy during brain magnetic resonance imaging (MRI) exams to help improve srove speed and reliability.

Can be used for our purposes? Tensorflow is the other popular framework other than PyTorch. It is widely used in production, but PyTorch is slowly catching up. It not as easy to work with, and a lot more researchers seem to be shifting to PyTorch. For this reason, we will not use it in our project.

Technological aspects:

Pros:

  • Built on top of Tensorflow, which is a powerful Deep Learning framework.

Cons:

  • Since it is built on top of Tensorflow, it can only be used for Deep Learning algorithms. Only a few simple ML models are available (e.g. linear regression and classification).
  • Not production ready, meaning that it can only simulate federated learning on a computer and cannot be run on a client-server system. (01/22/2021)

Later, TFF could be an option for using Deep Learning algorithms in federated way.

IntelLabs - OpenFL

https://github.com/IntelLabs/OpenFederatedLearning

Technological aspects:

Pros:

Cons:

  • Its development is in very early stage.
  • Since it is built on top of Tensorflow and PyTorch, it can only be used for Deep Learning algorithms.
  • Not production ready, it can only simulate federated learning on a computer and cannot be run on a client-server system. (01/22/2021)

IBM Federated Learning

https://ibmfl.mybluemix.net/github

Technological aspects:

Pros:

  • They are currently building the framework on top Tensorflow/Keras, PyTorch and Scikit-learn frameworks and libraries, so both simple ML models and Deep Learning models will be available.

Cons:

  • They have only a tech-preview of this framework, which is only available on IBM Cloud Pak for Data.

NVidia Clara Federated Learning

https://docs.nvidia.com/clara/tlt-mi/federated-learning/index.html

Pros:

  • It has a basic built-in authentication.

Cons:

  • Clara provides a training framework to help accelerate deep learning training and inference only for medical imaging use cases
  • Many things need to be configured in json files, e.g. model’s loss function and optimizer, and even parameters of model training, such as number of local epochs, participating clients, etc., as well as image path and corresponding labels, hence it is very inflexible. https://docs.nvidia.com/clara/tlt-mi/nvmidl/model.html
  • At client side, a person has to start the client code to be able to take part in a training round in case the admin stopped it remotely.
  • It uses synchronous gRPC communication between the server and clients.

In summary, it’s inflexible due to the lots of json configuration and since it can be used only for medical imaging use cases, it’s not suitable for our purposes.
